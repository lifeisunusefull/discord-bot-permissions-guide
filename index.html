Discord Bot Development: Permissions Guide

Overview

This guide provides a detailed explanation of the permission flags available for Discord bots. These permission flags are exported as BigInt values. Avoid converting them to numbers, as this may cause issues. Instead, use BigInts directly or use modules that support them.

Permission Flags

Each permission flag allows specific actions within Discord servers. Below is a list of available permission flags and their descriptions:

Channel-Specific Permissions

readonly CreateInstantInvite: bigint; // Allows creation of instant invites. Applies to: Text, Voice, Stage channels

readonly ManageChannels: bigint; // Allows management and editing of channels. Applies to: Text, Voice, Stage channels

readonly AddReactions: bigint; // Allows adding reactions to messages. Applies to: Text, Voice, Stage channels

readonly Stream: bigint; // Allows the user to go live. Applies to: Voice, Stage channels

readonly ViewChannel: bigint; // Allows guild members to view a channel, read messages in text channels, and join voice channels. Applies to: Text, Voice, Stage channels

readonly SendMessages: bigint; // Allows sending messages in a channel and creating threads in a forum (does not allow sending messages in threads). Applies to: Text, Voice, Stage channels

readonly SendTTSMessages: bigint; // Allows sending of `/tts` messages. Applies to: Text, Voice, Stage channels

readonly ManageMessages: bigint; // Allows deletion of other users' messages. Applies to: Text, Voice, Stage channels

readonly EmbedLinks: bigint; // Links sent by users with this permission will be auto-embedded. Applies to: Text, Voice, Stage channels

readonly AttachFiles: bigint; // Allows uploading images and files. Applies to: Text, Voice, Stage channels

readonly ReadMessageHistory: bigint; // Allows reading of message history. Applies to: Text, Voice, Stage channels

readonly MentionEveryone: bigint; // Allows using the `@everyone` tag to notify all users in a channel, and the `@here` tag to notify all online users in a channel. Applies to: Text, Voice, Stage channels

readonly UseExternalEmojis: bigint; // Allows using custom emojis from other servers. Applies to: Text, Voice, Stage channels

Guild-Wide Permissions

readonly KickMembers: bigint; // Allows kicking members.

readonly BanMembers: bigint; // Allows banning members.

readonly Administrator: bigint; // Allows all permissions and bypasses channel permission overwrites.

readonly ManageGuild: bigint; // Allows management and editing of the guild.

readonly ViewAuditLog: bigint; // Allows viewing of audit logs.

readonly PrioritySpeaker: bigint; // Allows using priority speaker in a voice channel. Applies to: Voice channels

readonly ViewGuildInsights: bigint; // Allows viewing guild insights.

readonly ManageRoles: bigint; // Allows management and editing of roles. Applies to: Text, Voice, Stage channels

readonly ManageWebhooks: bigint; // Allows management and editing of webhooks. Applies to: Text, Voice, Stage channels

readonly ManageGuildExpressions: bigint; // Allows editing and deleting emojis, stickers, and soundboard sounds created by all users.

readonly UseApplicationCommands: bigint; // Allows using application commands, including slash commands and context menu commands. Applies to: Text, Voice, Stage channels

readonly ManageEvents: bigint; // Allows editing and deleting scheduled events created by all users. Applies to: Voice, Stage channels

readonly ManageThreads: bigint; // Allows deleting and archiving threads, and viewing all private threads. Applies to: Text channels

readonly CreatePublicThreads: bigint; // Allows creating public and announcement threads. Applies to: Text channels

readonly CreatePrivateThreads: bigint; // Allows creating private threads. Applies to: Text channels

readonly ModerateMembers: bigint; // Allows timing out users to prevent them from sending or reacting to messages in chat and threads, and from speaking in voice and stage channels.

readonly ViewCreatorMonetizationAnalytics: bigint; // Allows viewing role subscription insights.

readonly UseSoundboard: bigint; // Allows using soundboard in a voice channel. Applies to: Voice channels

readonly CreateGuildExpressions: bigint; // Allows creating emojis, stickers, and soundboard sounds, and editing and deleting those created by the current user.

readonly CreateEvents: bigint; // Allows creating scheduled events, and editing and deleting those created by the current user. Applies to: Voice, Stage channels

readonly UseExternalSounds: bigint; // Allows using custom soundboard sounds from other servers. Applies to: Voice channels

readonly SendVoiceMessages: bigint; // Allows sending voice messages. Applies to: Text, Voice, Stage channels

readonly SendPolls: bigint; // Allows sending polls. Applies to: Text, Voice, Stage channels

Rate Limits

Discord enforces rate limits to prevent abuse of the API. If you exceed a rate limit, you will receive a response containing the following information:

interface RESTRateLimit {
 code?: number;
 global: boolean;
 message: string;
 retry_after: number;
}

Error Handling

When an error occurs, the API returns a JSON response containing details about the error. The structure of this response is as follows:

interface RESTError {
 code: number;
 message: string;
 errors?: RESTErrorData;
}
